O que é POO (Programação Orientada a Objetos)?

POO é um paradigma de programação baseado no uso de "objetos", que representam entidades do mundo real com características (atributos) e comportamentos (métodos).

Conceitos POO

  • Classe: É como um molde que define os atributos e comportamentos que os objetos terão.
  • Objeto: É uma instância da classe, ou seja, um exemplar real criado a partir do molde da classe.
  • Atributos: São as características ou propriedades de um objeto.
  • Métodos: São as ações ou comportamentos que os objetos podem executar.
  • Herança: Permite que uma classe herde atributos e métodos de outra, promovendo o reaproveitamento de código.
  • Polimorfismo: Permite que um mesmo método tenha comportamentos diferentes dependendo do contexto.
  • Encapsulamento: É o conceito de proteger os dados dentro de uma classe, controlando seu acesso.

Qual a vantagem de utilizar o POO?

  • Organização do código: Mais fácil de manter e entender.
  • Reutilização: Classes podem ser reutilizadas em vários projetos.
  • Facilidade de expansão: Adicionar novas funcionalidades sem mudar muito o código existente.
  • Modelagem do mundo real: Facilita o desenvolvimento de sistemas que refletem o comportamento de objetos reais.
